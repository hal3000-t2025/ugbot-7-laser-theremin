# 激光特雷门琴复现指南

这是一份面向第一次动手做电子项目的复现说明。

目标是让你从零开始，按步骤完成下面这件事：

- 接好硬件
- 把程序烧进板子
- 让琴发声
- 做一次正式校准
- 用按钮切换音色

如果你完全不懂电子，也可以照着做。不要跳步骤。

## 1. 先看最终要做成什么

做完以后，你会得到一台可以这样工作的“激光特雷门琴”：

- 一只手控制音高
- 另一只手控制音量
- 喇叭发声
- 按按钮切换预设
- 串口可以校准和查看状态

当前 `1.0` 版本的核心能力：

- `10` 个预设：
  - `1 sine`
  - `2 square`
  - `3 triangle`
  - `4 warm`
  - `5 hollow`
  - `6 bright`
  - `7 sample`
  - `8 sample2`
  - `9 example`
  - `10 example2`
- 音高范围默认 `220Hz - 1760Hz`
- 有保存校准功能
- 有硬件按钮切换预设功能

## 2. 你要准备哪些东西

先把这些东西放到桌上。

### 必需硬件

1. 主控板
   - `ESP32-S3 DevKitC-1 compatible N16R8`
   - 数量：`1`
   - 作用：运行整个项目

2. 数字功放板
   - `MAX98357A（已焊接排针）`
   - 数量：`1`
   - 作用：把数字音频变成喇叭能放的声音

3. 激光测距传感器
   - `VL53L1X`
   - 数量：`2`
   - 作用：
     - 第一个控制 `pitch`
     - 第二个控制 `volume`

4. 喇叭
   - `3W` 扬声器
   - 数量：`1`
   - 作用：发声

5. 硬件按钮
   - `DFRobot DFR0789 LED Switch`
   - 数量：`1`
   - 作用：切换预设

6. 杜邦线
   - 公对公 / 公对母，按你手头模块接口准备
   - 数量：越多越好

7. USB 数据线
   - 能传数据，不只是充电
   - 数量：`1`

### 建议准备

1. 面包板
   - 作用：方便接线

2. 标签纸或小纸条
   - 作用：给两只 `VL53L1X` 标记：
     - `pitch`
     - `volume`

3. 一台电脑
   - `macOS` / `Linux` 都可以
   - 这份指南按当前项目环境优先写 `macOS`

## 3. 先记住几个非常重要的规则

1. 所有模块先断电接线，不要边通电边乱插。
2. 这个项目当前所有小模块都按 `3.3V` 系统来接。
3. 喇叭黑线不要直接接开发板 `GND`，它要接功放板 `SPK-`。
4. 两只 `VL53L1X` 要分开接，不要把它们直接并到同一组 `SDA/SCL/XSHUT` 上。
5. 第一次复现时，不要额外加 OLED、旋钮、外壳，先把 1.0 基础版跑通。

## 4. 每个模块怎么接

下面只做最稳的 `1.0` 版本接线。

### 4.1 ESP32-S3 到 MAX98357A

- `ESP32 3V3` -> `MAX98357A VIN`
- `ESP32 GND` -> `MAX98357A GND`
- `ESP32 GPIO4` -> `MAX98357A BCLK`
- `ESP32 GPIO5` -> `MAX98357A LRC / WS`
- `ESP32 GPIO6` -> `MAX98357A DIN`
- `ESP32 3V3` -> `MAX98357A SD`

注意：

- `GAIN` 先不接
- `SD` 建议直接接 `3V3`
- 当前这套项目不要依赖开发板 `5V` 给功放供电
- 当前稳定做法就是 `3V3 -> VIN`

### 4.2 喇叭怎么接

- `3W` 扬声器红线 -> `MAX98357A SPK+`
- `3W` 扬声器黑线 -> `MAX98357A SPK-`

不要这样接：

- 喇叭黑线 -> `ESP32 GND`

这是错的。

### 4.3 Pitch 传感器怎么接

把第一只 `VL53L1X` 当作 `pitch` 传感器。

- `ESP32 3V3` -> `VL53L1X VIN / VCC`
- `ESP32 GND` -> `VL53L1X GND`
- `ESP32 GPIO8` -> `VL53L1X SDA`
- `ESP32 GPIO9` -> `VL53L1X SCL`
- `ESP32 GPIO13` -> `VL53L1X XSHUT`

如果模块上还有下面这些脚：

- `INT`
- `GPIO1`
- `IRQ`

这几个先不接。

### 4.4 Volume 传感器怎么接

把第二只 `VL53L1X` 当作 `volume` 传感器。

- `ESP32 3V3` -> `VL53L1X VIN / VCC`
- `ESP32 GND` -> `VL53L1X GND`
- `ESP32 GPIO11` -> `VL53L1X SDA`
- `ESP32 GPIO12` -> `VL53L1X SCL`
- `ESP32 GPIO14` -> `VL53L1X XSHUT`

同样地：

- `INT`
- `GPIO1`
- `IRQ`

先不接。

### 4.5 按钮怎么接

- `ESP32 3V3` -> `DFR0789 +`
- `ESP32 GND` -> `DFR0789 -`
- `ESP32 GPIO16` -> `DFR0789 D`

这个按钮接好后，按一下会切到下一个预设。

## 6. 软件环境怎么准备

### 6.1 安装 Python

先确认电脑有 `Python 3`：

```bash
python3 --version
```

如果能看到版本号，就可以继续。

### 6.2 安装 PlatformIO

如果你还没有 `pio` 命令，就安装它：

```bash
python3 -m pip install -U platformio
```

安装完确认一下：

```bash
pio --version
```

### 6.3 进入项目目录

```bash
cd /Users/houtao/ai/Mcp_Dev/激光特雷门琴
```

### 6.4 建议使用项目虚拟环境

如果项目里已经有 `.venv`，先激活：

```bash
. .venv/bin/activate
```

如果没有，也可以直接用系统里的 `pio`。

## 7. 第一次编译

在项目目录运行：

```bash
. .venv/bin/activate
pio run -e esp32-s3-devkitc-1
```

如果成功，你会看到最后有：

```text
[SUCCESS]
```

## 8. 找到你的串口

先把板子用 USB 线连上电脑，然后运行：

```bash
python3 tools/theremin_tool.py ports
```

你会看到一串设备名，比如：

```text
/dev/cu.usbmodem1101
```

你后面所有命令里，把 `--port` 换成你自己的串口名。

## 9. 把程序烧进板子

假设你的串口是：

```text
/dev/cu.usbmodem1101
```

就运行：

```bash
. .venv/bin/activate
pio run -e esp32-s3-devkitc-1 -t upload --upload-port <你的串口>
```

如果烧录成功，也会看到：

```text
[SUCCESS]
```

## 10. 查看启动日志

烧录后，打开串口监视：

```bash
. .venv/bin/activate
pio device monitor -b 115200 --port <你的串口>
```

正常情况下，你应该看到类似这些内容：

```text
Laser Theremin M5 boot
Board: ESP32-S3 DevKitC-1 compatible N16R8
Test tone: 440.0 Hz
Pitch I2C ready on SDA=8 SCL=9 XSHUT=13
Volume I2C ready on SDA=11 SCL=12 XSHUT=14
I2S init ok, streaming sine wave.
Pitch VL53L1X init ok at 0x29.
Volume VL53L1X init ok at 0x29.
```

如果你看到传感器 `init failed`，先不要怪软件，先回头查接线。

## 11. 用串口工具做第一次检查

先关闭任何已经打开的串口监视器，然后运行：

```bash
python3 tools/theremin_tool.py --port <你的串口> send --quiet-stream status
```

你应该能看到当前预设、音高、音量、校准参数等状态。

然后做一次快速自检：

```bash
python3 tools/theremin_tool.py --port <你的串口> smoke-test
```

这一步会自动切换：

- `warm`
- `hollow`
- `bright`
- `sample`
- `sample2`
- `example`
- `example2`
- 再回到 `sine`

如果这一步能走完，说明：

- 串口通信正常
- 预设切换正常
- 板子程序正常运行

## 12. 正式校准

这是最重要的一步。

先运行：

```bash
python3 tools/theremin_tool.py --port <你的串口> calibrate --save
```

程序会依次让你做 4 次动作：

1. `pitch near`
2. `pitch far`
3. `volume near`
4. `volume far`

每一次都记住这一句：

- 先把手摆好
- 再按回车
- 按回车后保持不动

### 12.1 pitch near

- 把手放到 `pitch` 传感器前面比较近的位置
- 推荐先试 `6 到 10 cm`
- 这是高音区

### 12.2 pitch far

- 把手移远
- 推荐先试 `35 到 50 cm`
- 这是低音区

### 12.3 volume near

- 把手放到 `volume` 传感器前较近的位置
- 推荐 `6 到 10 cm`
- 这是大音量端

### 12.4 volume far

- 把手从 `volume` 传感器前移开
- 推荐 `40 到 60 cm`
- 这是静音或接近静音的一端

完成后程序会自动 `save`。

## 13. 校准完成后怎么检查

运行：

```bash
python3 tools/theremin_tool.py --port <你的串口> send --quiet-stream status
```

重点看这一行：

```text
cal pitch near=... far=... filter=... curve=... | volume near=... far=... smooth=... gate=... max=...
snap scale=... root=... width=... strength=... smooth=...
```

你至少要满足：

- `pitch near < pitch far`
- `volume near < volume far`

## 14. 怎么开始演奏

### 普通演奏模式

默认你可以用：

- 一只手控制 `pitch`
- 另一只手控制 `volume`

### 自动播放模式

切到 `example` 预设后：

- `pitch` 传感器控制播放速度
- `volume` 传感器控制音量
- `pitch` 手靠近时更慢
- `pitch` 手离远时更快

切到 `example2` 预设后：

- 逻辑和 `example` 一样
- 曲目是《天空之城》主旋律
- 当前版本已经整体升高 `1` 个八度

注意：

- 自动播放预设不是“自己固定音量外放”
- 如果你的手没有进入 `volume` 检测区，曲子会继续播放但保持静音

## 15. 按钮怎么用

按钮接好后：

- 每按一次
- 切到下一个预设

顺序是：

1. `sine`
2. `square`
3. `triangle`
4. `warm`
5. `hollow`
6. `bright`
7. `sample`
8. `sample2`
9. `example`
10. `example2`

然后再回到 `1`

## 17. 如果没有声音，先查什么

按这个顺序查：

1. USB 线是不是数据线
2. `MAX98357A VIN` 有没有接到 `3V3`
3. `MAX98357A SD` 有没有接到 `3V3`
4. `BCLK / LRC / DIN` 有没有接错
5. 喇叭是不是接到了 `SPK+ / SPK-`
6. 两只 `VL53L1X` 的 `SDA / SCL` 有没有接反
7. `XSHUT` 有没有接对
8. 有没有只开了串口监视器但没真正烧录成功
9. 当前是不是 `mute`
10. `volume` 手是不是离得太远，导致当前几乎静音
11. 如果当前在 `example / example2` 自动播放模式，手有没有进入 `volume` 检测区

## 18. 如果传感器不工作，先查什么

1. `VCC` 和 `GND` 有没有接反
2. 是不是把两只 `VL53L1X` 接到了同一组 `XSHUT`
3. `pitch` 和 `volume` 的线有没有插反
4. 模块上的 `INT` 有没有误接到别的地方
5. 启动日志里到底是哪一只传感器失败

## 19. 最终验收清单

如果下面 6 条都通过，就说明你复现成功了。

1. 板子能正常上电
2. 串口能看到启动日志
3. 两只 `VL53L1X` 都能初始化成功
4. 喇叭能发声
5. `smoke-test` 能跑完
6. `calibrate --save` 后可以正常演奏

## 20. 当前 1.0 版本建议

为了让复现更稳：

- 第一轮不要加 OLED
- 第一轮不要加旋钮
- 第一轮不要上外壳
- 先把基础版跑通

等基础版稳定后，再往上叠加扩展件。
