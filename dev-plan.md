先定边界：

* 主控：**ESP32-S3 DevKitC-1**
* 传感器：**2 × VL53L1X**
* 音频：**MAX98357A + 8Ω 小喇叭**
* 开发环境：**PlatformIO + Arduino 框架**
* 第一目标：先做成**可演奏的数字特雷门原型**
* 第二目标：再加**音色切换 / wavetable**
* 第三目标：最后再碰 **WAV sample**
  ESP32-S3 DevKitC-1 是 Espressif 官方开发板，PlatformIO 也提供了 Espressif32 平台支持；VL53L1X 和 MAX98357A 都有成熟资料可参考。([Espressif Systems][1])

---

## 一、总开发路线

### Phase 0：硬件到货与最小连线确认

目标不是写功能，而是确认板子和模块都活着。

Codex 任务：

* 建立 PlatformIO 工程
* 点亮板载串口日志
* 跑一个最小 `main.cpp`
* 接好 I2C 和 I2S 引脚定义
* 输出开机日志，确认串口工作

验收标准：

* 串口稳定输出启动信息
* 能成功编译、烧录、重启
* 平台配置固定下来，不再反复改板型

---

### Phase 1：先把音频链路打通

这一步只关心：**ESP32-S3 能不能稳定发声**。

Codex 任务：

* 使用 I2S 驱动 MAX98357A
* 输出固定频率测试音，例如 440Hz 正弦波
* 加入音量参数
* 保证持续播放无明显爆音、断音

为什么先做这个：
MAX98357A 接收标准 I2S 数字音频，适合直接由 ESP32-S3 驱动；它面向 4Ω 及以上扬声器，8Ω 喇叭可以用，只是声压通常会比 4Ω 小一点。([Adafruit 學習系統][2])

验收标准：

* 上电后持续发出稳定测试音
* 改频率参数时，音高明显变化
* 改音量参数时，声音明显变化
* 运行 10 分钟不死机

---

### Phase 2：单传感器控制 Pitch

这一步开始接入手势，但只上一个传感器，降低复杂度。

Codex 任务：

* 接入 1 个 VL53L1X
* 通过 I2C 读取距离
* 做基本滤波：移动平均或指数平滑
* 把距离映射成频率
* 距离越近，音高越高；距离越远，音高越低

VL53L1X 有现成 Arduino 库，先用成熟库，不要自己硬啃底层寄存器。([GitHub][3])

验收标准：

* 手在有效区域内移动时，音高连续变化
* 不需要极精确，但不能明显抽搐
* 无手时进入静音或默认频率

---

### Phase 3：双传感器控制 Pitch + Volume

这一步才是“像特雷门”。

Codex 任务：

* 接入第 2 个 VL53L1X
* 解决两个同地址传感器初始化问题
* 右手传感器控制 Pitch
* 左手传感器控制 Volume
* 双手同时操作时，声音稳定

这里的重点不是功能，而是**系统设计**：
要把“传感器采样”和“音频输出”分离，避免因为 I2C 读取卡住 I2S 音频。VL53L1X 是 I2C 设备，MAX98357A 走 I2S，这种分层非常适合拆成独立模块。([GitHub][3])

验收标准：

* 右手改音高，左手改音量，彼此互不干扰
* 双手同时移动时，没有明显破音或卡顿
* 参数更新频率足够快，手感自然

---

### Phase 4：基础音色引擎

先不要碰 WAV，先做“真正可控的音色层”。

Codex 任务：

* 抽象一个 `SynthEngine`
* 支持 3 种基础波形：

  * sine
  * square
  * triangle
* 支持按钮或串口命令切换音色
* 支持参数平滑，避免切换时爆音

这一阶段的目标是：
你已经有一个**真正能演奏的数字乐器核心**，而不是“传感器+蜂鸣器”。

验收标准：

* 能切换 3 种波形
* 每种波形都可被双手实时控制
* 切换时系统不崩，不出现长时间静音

---

### Phase 5：wavetable 音色

这是你真正开始进入“自定义音色”的阶段。

Codex 任务：

* 实现相位累加器
* 支持单周期 wavetable
* 提供 2–3 个内置波表
* 保留 Pitch/Volume 控制链路不变
* 允许后期把你自己的波形数组塞进去

这一步比 WAV sample 更适合先做，因为它保留了乐器手感，又比整段 WAV 简单得多。

验收标准：

* 波表音色能稳定发声
* 调 pitch 时声音仍然连续
* 更换波表不影响手势控制稳定性

---

### Phase 6：短 WAV sample

这是高阶目标，放在最后。

Codex 任务：

* 支持读取 flash 内置短样本，暂不优先上 SD 卡
* 实现简单重采样
* 最好加线性插值
* 支持 loop 区域
* 把短样本当作“可演奏音色”而不是完整音乐播放

验收标准：

* 短样本可以被连续演奏
* 变调时不至于完全崩坏
* 延迟和连续性仍然可接受

---

## 二、建议的代码结构

让 Codex 按这个结构写，不要写成单文件地狱。

```text
src/
  main.cpp
  app_config.h
  pins.h

  audio/
    audio_engine.h
    audio_engine.cpp
    oscillator.h
    oscillator.cpp
    wavetable_engine.h
    wavetable_engine.cpp

  sensors/
    tof_sensor.h
    tof_sensor.cpp
    hand_mapper.h
    hand_mapper.cpp

  control/
    theremin_controller.h
    theremin_controller.cpp
    smoothing.h

  ui/
    serial_debug.h
    serial_debug.cpp
```

核心原则：

* `audio/` 只负责发声
* `sensors/` 只负责采样和预处理
* `control/` 只负责映射逻辑
* `main.cpp` 只负责组装系统

## 三、建议的里程碑

你可以要求 Codex 按里程碑提交，而不是一次做完。

### M1：板子能发固定音

交付：

* PlatformIO 工程
* I2S 初始化
* 固定 440Hz 正弦波

### M2：一个传感器控制音高

交付：

* 传感器读取
* 平滑滤波
* Pitch 映射

### M3：两个传感器完整演奏

交付：

* 双 VL53L1X
* Pitch + Volume
* 串口调试信息

### M4：音色切换

交付：

* sine/square/triangle
* 可切换

### M5：wavetable

交付：

* 2–3 个波表
* 可扩展接口

### M6：sample playback

交付：

* 短样本
* 简单重采样
* loop

## 四、你可以直接交给 Codex 的开发要求

下面这段你几乎可以原样发给它：

```text
为 ESP32-S3 DevKitC-1 创建一个 PlatformIO 项目，使用 Arduino 框架。
项目目标是实现一个基于双 VL53L1X 和 MAX98357A 的数字特雷门原型。

硬件：
- ESP32-S3 DevKitC-1
- 2 x VL53L1X ToF 传感器
- 1 x MAX98357A I2S 功放
- 1 x 8Ω 小喇叭

开发阶段要求按里程碑推进，不要一次实现全部功能。

M1:
- 初始化串口
- 初始化 I2S
- 通过 MAX98357A 输出稳定的 440Hz 正弦波
- 代码拆分为 audio 模块和 main.cpp

M2:
- 接入 1 个 VL53L1X
- 稳定读取距离并输出到串口
- 将距离映射到音高
- 做基本平滑，避免音高剧烈跳变

M3:
- 接入第 2 个 VL53L1X
- 解决双传感器初始化和地址问题
- 一个传感器控制 pitch，一个控制 volume
- 音频线程和传感器读取逻辑解耦

M4:
- 支持 sine / square / triangle 三种基础波形
- 支持串口命令切换波形

要求：
- 保持模块化结构
- 不要写成单文件
- 每个阶段都给出编译说明、接线说明、验收方法
- 优先保证稳定性和实时性，不要过早引入 WAV 或 SD 卡
```

## 五、你买到硬件后，先让 Codex做哪一版

最正确的顺序不是“大而全”，而是这三步：

先让它做 **M1**。
M1 成功后再做 **M2**。
M2 手感对了再做 **M3**。

不要让它一开始就：

* 双传感器
* WAV
* MQTT
* UI
* OLED
* Wi-Fi
  全部一起上。

这种写法大概率会烂尾。

## 六、我对你这个项目的额外建议

### 1. 优先买 VL53L1X，不是 VL53L0X

如果价格差别不大，VL53L1X 更适合这个项目路线。社区里也有成熟 Arduino 库。([GitHub][3])

### 2. 第一版不要上 SD 卡

因为真正难点不是“读文件”，而是“连续可演奏音频”。

### 3. 第一版不要追求“像传统特雷门”

先追求：

* 稳定
* 跟手
* 可重复

### 4. 第一版只做桌面原型

外观和“假天线”可以放后面。

---

## 七、最终版项目目标

你可以把最终目标定义成这样：

> 一个基于 ESP32-S3 的数字特雷门原型，使用双 VL53L1X 进行手势测距，使用 MAX98357A 输出实时音频，先实现基础波形演奏，再扩展到 wavetable，最后支持短 WAV 采样音色。
